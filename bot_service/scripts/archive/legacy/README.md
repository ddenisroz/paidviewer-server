# Legacy-скрипты

Здесь лежат старые и destructive-сценарии, которые не должны использоваться в обычном workflow.

## Что сюда относится

- полная очистка базы
- reset сценарии
- старые миграционные helper-скрипты, оставленные только как архив

## Правило

- сначала использовать preview/dry-run, если он есть
- не держать такие скрипты в корне `bot_service/scripts/`
- не запускать их без осознанного понимания последствий

Если скрипт снова становится рабочим operational-инструментом, его нужно переписать и вернуть из архива в нормальный maintenance-слой.
